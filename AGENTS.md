# JalanKita ‚Äî AGENTS.md

Internal road damage reporting app for **Dinas PU Bina Marga Kabupaten Sidoarjo**. Three roles: `petugas`, `supervisor`, `admin` (not yet implemented). No public/warga role.

## Stack & layout

```
Frontend/               React 19 + TanStack Start SSR + Tailwind v4, Vite
backend_POSTGRESQL/     Laravel 13 REST API (PHP 8.3, Sanctum)
backend_AI/             FastAPI + YOLOv8s (4 damage classes, port 8000)
scripts/                start-dev-with-ngrok.sh (all 3 servers + tunnel)
```

Frontend hosts on `:5173`. Vite proxy forwards `/api/*` ‚Üí `http://localhost:8080` (Laravel). Laravel forwards AI requests to FastAPI at `:8000`.

## Dev commands

```bash
# Frontend
npm install && npm run dev          # Vite on :5173

# Laravel
composer install && php artisan serve --port=8080
# Or: composer run dev  # starts artisan :8080 + queue + logs + Vite concurrently

# Frontend checks

```

## Key structural facts

- **`vite.config.ts`** uses `@lovable.dev/vite-tanstack-config` ‚Äî do NOT add Vite plugins manually (TanStack Start plugin, React plugin, Tailwind, etc. are already bundled).
- **`routeTree.gen.ts`** is auto-generated ‚Äî do not edit. Regenerated by TanStack Router codegen.
- **`src/server.ts`** is the SSR entrypoint (referenced by `wrangler.jsonc` and vite config).
- **`src/lib/auth.ts`** stores token + user in `localStorage` (known XSS risk, not fixed).
- **`solution.md`** is the original implementation spec (trust score, batch upload, autocomplete guarding) ‚Äî preserved as reference. Do not delete.
- **`SECURITY_ANALYSIS.md`** documents known security issues vs fixed items.

## API routes (`backend_POSTGRESQL/routes/api.php`)

| Endpoint | Auth | Notes |
|---|---|---|
| `POST /auth/login` | Public | Throttled 10/min |
| `POST /analyze` | Sanctum | Single photo AI analysis (forward to FastAPI) |
| `POST /reports` | Sanctum | Single report store |
| `POST /analyze-batch` | Sanctum | Batch AI (max 20 photos) |
| `POST /reports/batch` | Sanctum | Batch report store (main + sub-reports) |
| `GET /v1/reports/check-duplicate` | Public | Spatial (15m Haversine) + textual dedup |

All critical validation (EXIF date, GPS EXIF extraction, road name vs coordinate matching, trust score) is in backend ‚Äî frontend is UX layer only.

## Non-obvious conventions

- **Trust score is triage, not gatekeeper** ‚Äî never hard-reject a report because score is low. Supervisor is final decision maker.
- **Batch upload**: each sub-report gets its own coordinate from GPS EXIF if available, falls back to form coordinate. Photos without EXIF date/warning are skipped but don't fail the batch.
- **Coordinate boundary**: all coordinates validated to be within Sidoarjo / Indonesia bounds.
- **EXIF date validation**: photos > 2 days old or future-dated are rejected for single upload; in batch they are skipped per-file.
- **Image dedup**: SHA-256 hash on file content. Duplicate photos within or across batches are silently skipped.
- **Road name**: must be selected from autocomplete dropdown (LocationIQ). Manual typing is warned in UI but not blocked at API level ‚Äî trust score handles it.
- **Always add new DB columns via migration** ‚Äî never modify existing columns or tables.
- **Store photos on disk** (`storage/app/public/`) ‚Äî never as base64 in DB.
