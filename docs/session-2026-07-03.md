## B. ERR_BLOCKED_BY_ORB (Gambar Rusak di Supervisor)

### Root Cause

Laravel 13 **menghapus** `forceRootUrl()` dari `RoutingServiceProvider`. `asset()` sekarang fallback ke `request->root()` yang membaca `Host` header. Vite proxy dengan `changeOrigin: true` mengirim `Host: localhost:8080`, jadi `asset()` return URL `:8080` — cross-origin dari `:5173`.

Rantai lengkap:
1. APP_URL di `.env` masih `http://localhost:8080`
2. `asset('storage/...')` → `http://localhost:8080/storage/...` (cross-origin)
3. Gambar tidak ditemukan → Laravel return 404 HTML
4. Service Worker **CacheFirst** menyimpan 404 HTML ke cache (berlaku 30 hari)
5. Chrome ORB blokir response `text/html` yang di-serve sebagai `<img>`
6. Setelah server diperbaiki, SW tetap sajikan 404 HTML dari cache

### Fix Layer-by-Layer

| Lapisan | Masalah | Fix | File |
|---|---|---|---|
| Asset URL | `asset()` pakai Host header (`:8080`) | `URL::useOrigin(config('app.url'))` di `AppServiceProvider::boot()` | `app/Providers/AppServiceProvider.php:26` |
| Service Worker | CacheFirst simpan 404 HTML | `NetworkFirst` + cache name `images-v2` + hapus cache `-v1` | `src/sw.ts` |
| TanStack Query | Cache data lama dengan URL salah | `invalidateQueries` on mount + `refetchOnMount: 'always'` | `src/routes/supervisor/index.tsx` |
| VitePWA | Gagal inject registrasi SW di dev mode | Manual `navigator.serviceWorker.register('/sw.js')` | `src/client.tsx` |
| Vite proxy | Cross-origin tanpa safety net | Inject `Access-Control-Allow-Origin: *` header | `vite.config.ts` |

### File Diubah

| File | Perubahan |
|---|---|
| `backend_POSTGRESQL/.env` | `APP_URL=http://localhost:5173` |
| `backend_POSTGRESQL/app/Providers/AppServiceProvider.php:26` | `URL::useOrigin(config('app.url'))` |
| `Frontend-stable/src/sw.ts` | `NetworkFirst`, cache `images-v2`, OLD_CACHE_PATTERNS, activate cleanup |
| `Frontend-stable/src/client.tsx` | Manual SW registration |
| `Frontend-stable/src/routes/supervisor/index.tsx` | TanStack Query invalidation + refetchOnMount |
| `Frontend-stable/vite.config.ts` | CORS headers safety net on `/storage` proxy |

## C. 403 Forbidden di Storage Images

### Root Cause

PHP built-in server (`php artisan serve`) di Windows memiliki security feature: **tidak mengikuti symlink**. Saat request `/storage/reports/...jpg` masuk, PHP melihat direktori `public/storage` adalah symlink ke `storage/app/public`, dan langsung return **403 Forbidden** tanpa mencoba mencari file.

Ini berbeda dengan 404 — 403 tidak memberikan informasi apakah file ada atau tidak. Chrome ORB tidak bisa membedakan.

### Fix

Tambah route fallback di `routes/web.php` yang intercept `/storage/{path}` dan serve langsung dari `storage_path('app/public/')`:

```php
if (app()->environment('local')) {
    Route::get('/storage/{path}', function (string $path) {
        $fullPath = storage_path('app/public/'.$path);
        if (!file_exists($fullPath)) {
            abort(404);
        }
        return response()->file($fullPath);
    })->where('path', '.*');
}
```

Route ini hanya aktif di environment `local`. Guarded supaya tidak mengganggu production (Nginx/Apache handle langsung).

| File | Perubahan |
|---|---|
| `backend_POSTGRESQL/routes/web.php:11-21` | Route fallback `/storage/{path}` |

## E. Pelajaran & Konteks untuk AI Berikutnya

1. **Laravel 13 `asset()` behavior**: Laravel 13 menghapus `forceRootUrl()` dari `RoutingServiceProvider`. `asset()` sekarang fallback ke `request->root()` yang membaca `Host` header. Jika ada proxy dengan `changeOrigin: true`, asset URL akan pakai port backend, bukan frontend. Fix: `URL::useOrigin(config('app.url'))` di `AppServiceProvider`.

2. **PHP built-in server + symlinks di Windows**: Selalu return 403, bukan 404, untuk file di symlinked directories. Tidak bisa di-fix tanpa route fallback. Route fallback harus di guard dengan `app()->environment('local')`.

3. **Service Worker caching strategy**: `CacheFirst` untuk gambar berbahaya di dev karena bisa menyimpan error response. `NetworkFirst` lebih aman. Selalu ganti cache name saat ubah strategy supaya SW lama tidak serve cached error.

4. **Android EXIF GPS**: Android 13+ Photo Picker redacts GPS EXIF di OS level. Telegram juga strips EXIF GPS dari foto. `Message.location` adalah satu-satunya sumber GPS yang reliable untuk Telegram Bot.

5. **Telegram API `reply_markup`**: Jangan kirim key `reply_markup` jika null. Telegram API return error 400 "object expected".

6. **VitePWA injectManifest dev mode**: Tidak meng-inject registrasi script ke HTML di dev mode dengan TanStack Start. Registrasi SW harus manual di `client.tsx`.

7. **`isNativePlatform()`**: `typeof window.Capacitor !== 'undefined'` true bahkan di browser (side effect import `@capacitor/core`). Selalu pakai `window.Capacitor.isNativePlatform?.() === true`.

8. **Report status untuk warga**: Jangan pakai `Menunggu Verifikasi` (dead-end, tidak ada yang review). Pakai `Menunggu Review` langsung — masuk ke tab supervisor "Perlu Review".
